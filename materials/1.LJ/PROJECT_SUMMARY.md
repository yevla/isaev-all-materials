# Подробный технический отчет: Классификация и организация архива ЖЖ Дмитрия Исаева

Данный документ содержит детальное описание всех этапов обработки данных блога `dsisaev.livejournal.com`, реализованных алгоритмов и структуры полученных знаний.

## 1. Конвейер обработки данных (Data Pipeline)

### Этап 1: Сбор данных (Scraping)
*   **Инструмент**: `src/scraper.py` (на базе `requests` и `BeautifulSoup`).
*   **Логика**: Сбор ссылок из `links.txt`, обход каждой страницы с обработкой различных версий разметки LiveJournal (старый и новый дизайн).
*   **Deduplication**: Реализована проверка по URL-адресу, чтобы избежать повторного сбора одних и тех же постов при перезапусках.
*   **Результат**: Файл `data/raw_posts.csv` (279 записей).

### Этап 2: Очистка и нормализация (Data Cleaning)
*   **Инструмент**: `src/clean_data.py`.
*   **Действия**:
    *   Удаление HTML-тегов, JS-скриптов и лишних пробелов.
    *   Нормализация дат публикаций.
    *   Фильтрация пустых или технических постов.
    *   *Decision*: По требованию пользователя были исключены метрики вовлеченности (лайки, репосты), так как они не несли семантической ценности для архива.

### Этап 3: Создание профессиональной таксономии
*   **Локация**: `config/categories.txt`.
*   **Структура**: Двухуровневая система.
    *   *Категории (7)*: Психология, Медицина, Личное, Места, Культура, Социум, Стиль.
    *   *Теги*: Сотни специфических меток, разделенных на группы:
        *   **Клинические типы**: Депрессия, Шизофрения, Алкоголизм.
        *   **Сматические типы**: Коронар, Язва, Астматик, Лишний Вес, Онкология.
        *   **Сценарные типы**: Будь Совершенным, Старайся, Радуй Других.
        *   **Путешествия**: Израиль, Италия, Франция, Одиссея.

## 2. Процесс классификации (Classification)

Классификация проводилась в **14 циклов (батчей)** для обеспечения максимальной точности.
*   Каждый пост (с 1 по 279) был проанализирован на предмет соответствия таксономии.
*   **Особое внимание**: Сохранена связь между личным опытом автора и профессиональными выводами. Посты, начинающиеся как «бытовые», часто переходят в глубокий психологический анализ.
*   **Итог**: `data/final_posts.csv` — центральный файл проекта, содержащий структурированные знания.

## 3. Интерактивные инструменты (Tools)

### Тематический Dashborad (`dashboard/index.html`)
Создан как альтернатива электронным таблицам для подготовки книги.
*   **Технологии**: Vanilla JS, CSS Glassmorphism.
*   **Возможности**:
    *   Мгновенная фильтрация по категориям (Sidebar).
    *   Поиск по тегам «в реальном времени».
    *   Full-text Overlay: интерфейс чтения с типографикой, оптимизированной под лонгриды (шрифт Lora).
    *   Статистика: счетчик постов в каждой выборке.

### Аналитический модуль (`src/analyze_overlap.py`)
Инструмент для понимания «карты мыслей» автора.
*   **Вывод**: Построена матрица пересечений. Самая сильная корреляция: `Психология + Личное` (53 поста), что подтверждает терапию через проживание.
*   **Идентификация**: Выделены «Чистые» профессиональные блоки для точечного анализа.

## 4. Экспорт для NotebookLM (Semantic AI)

Для работы с AI-системой NotebookLM данные были преобразованы из табличного вида в семантически связанные Markdown-файлы.

### Логика разделения (Chunking Strategy):
1.  **Professional_Core.md**: Весь экспертный контент. Сюда включена вся категория **Медицина**, так как анализ показал, что описания «соматических типов» и врачебной практики являются ядром знаний автора.
2.  **Professional_Applied_Living.md**: Психология в контексте повседневной жизни и семьи.
3.  **Professional_Applied_Travel.md**: Цикл «Одиссея» — психологические портреты мест и наций.
4.  **Societal_Psychology.md**: Психология больших групп, политика и социум.
5.  **Lifestyle_Culture.md**: Культурный контекст, стихи, юмор.

### Особенности формата экспорта:
*   В каждый пост вшиты метаданные (`**Категории**`, `**Теги**`). Это позволяет NotebookLM выдавать ответы с цитированием конкретных категорий знаний.
*   Удалены «анонсы» и технические уведомления короче 150 символов для исключения шума.
